## How can I afford to, and why do I try so hard to build the app for so many platforms?

The answer is rather disappointing: I didn’t have to try “that hard”.

**Most of the codes are reusable**. The main app is the sidebar Chrome extension written in React. The rest of the platforms use web view and iframe with some small custom logic to handle platform-specific requirements.

**Some technical stuff**

If you are looking to write an extension for multiple browsers, I highly recommend using this boilerplate.

[https://github.com/abhijithvijayan/web-extension-starter](https://github.com/abhijithvijayan/web-extension-starter?utm_campaign=Tony%20Dinh%E2%80%99s%20Newsletter&utm_medium=email&utm_source=Revue%20newsletter)

It supports Chrome, Firefox, Edge, Brave, Opera out of the box. Very convenience.

For Safari extension, check out [this article](https://developer.apple.com/documentation/safariservices/safari_web_extensions/converting_a_web_extension_for_safari?utm_campaign=Tony%20Dinh%E2%80%99s%20Newsletter&utm_medium=email&utm_source=Revue%20newsletter). Apple allows you to reuse your Chrome extension with very little code.

Good luck!

## I built a small product: EmojiAI.com

I had access to OpenAI’s API some time ago. I used the playground a lot but didn’t have time to actually build any product around it.

Until now!

<https://www.producthunt.com/products/emojiai-com>

It’s a small product, solving a small problem, but it helped me to see the process of building with OpenAI/GPT-3.

**Fun fact: I built this product in 1 day!**

1. Design and test [GPT-3 prompts](https://beta.openai.com/docs/guides/completion/prompt-design?utm_campaign=Tony%20Dinh%E2%80%99s%20Newsletter&utm_medium=email&utm_source=Revue%20newsletter) using OpenAI’s playground.
2. Once I’m happy with the prompts, I put the code in a Cloudflare Worker, that’s my entire “backend”.
3. Create a project with Next.JS for the frontend and call the API.
4. Register a domain and deploy the frontend using Cloudflare Pages.
5. And the last step to truly “complete” the project: **Add payment**.
6. Also, I need to request OpenAI’s approval of the release of the app to the public. The review process took less than 24 hours, which is very fast!

## Some thoughts

At this MRR; the one-off revenue from [DevUtils.app](https://devutils.app?utm_campaign=Tony%20Dinh%E2%80%99s%20Newsletter&utm_medium=email&utm_source=Revue%20newsletter); and other income sources, I can live comfortably almost anywhere in the world.

Circumstances have not allowed me to travel much this year, but I’ll do it as soon as I can.

I can’t retire yet, but the short-term financial independence allows me to be less stressed about things I don’t enjoy doing (like doing marketing 😂) and spend more time doing what I love.

I love building software and seeing people use it. I’ve been doing it [since I know how to program](https://www.softpedia.com/get/Security/Encrypting/All-File-Locker-Pro.shtml?utm_campaign=Tony%20Dinh%E2%80%99s%20Newsletter&utm_medium=email&utm_source=Revue%20newsletter), and now have the freedom to do it again!

## My new goal

My target now is to make [DevUtils](https://devutils.app?utm_campaign=Tony%20Dinh%E2%80%99s%20Newsletter&utm_medium=email&utm_source=Revue%20newsletter) the must-have app for all developers (like Chrome DevTool!), and make [Black Magic](https://blackmagic.so?utm_campaign=Tony%20Dinh%E2%80%99s%20Newsletter&utm_medium=email&utm_source=Revue%20newsletter) the best companion app for all Twitter power users (like TweetDeck!).

Having more revenue is nice, but chasing that metric **too closely** will often lead to some “not-so-good” outcomes (e.g., dark patterns).

**From 5K MRR onwards, I want to focus more on building great products, rather than growing fast.**

Since I’m an active user of both of my products, I’m pretty sure I’ll benefit a lot from this new goal! 😂 (Also true for all of my existing customers.)

I think it’s more important to keep the existing customer happy than to seek more customers, especially for subscription-based products.
